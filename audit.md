## Architecture & Design
- Modules such as Config, Db, Users, Todos, Admin, Settings, Attachments, OCR, and Workflows are all registered directly in `AppModule`, which keeps the dependency graph explicit but forces eager loading of every feature (apps/api/src/app.module.ts:1); consider grouping related sets behind feature modules or lazy loading to lower coupling and shorten bootstrap time.
- The `TodosService` handles creation, scheduling, parent-child validation, stage transitions, bulk mutations, and association helpers in a single class (`create`, `schedule`, `update`, `remove`, `associateTask`, etc.), mixing unrelated concerns and making it harder to test or extend (apps/api/src/todos/todos.service.ts:170;419;290); delegating scheduling and relationship enforcement to smaller collaborators would improve cohesion.
- List responses enrich each todo with child counts by running `getChildCount` per row inside `Promise.all`, issuing an additional query per task after every fetch (apps/api/src/todos/todos.service.ts:64;167); this N+1 pattern causes latency to grow with the result set, so replacing it with one aggregate query (e.g., `COUNT(*) FILTER`) would keep list performance predictable.
- Client-side hooks (`useAuth`, `useTodos`, `useScheduledEvents`) share state and dispatch `notifyTodoUpdate` to keep tabs synchronized, which gives every page a single source of truth despite being a client-heavy SPA (apps/web/app/hooks/useTodos.ts:60;364).

## Security & Reliability
- `setAuthCookie` defaults `secure` to `false` whenever `COOKIE_SECURE` is absent, so JWT sessions are delivered over plain HTTP if the env var is unset even though the token is `httpOnly` (apps/api/src/auth/auth.controller.ts:171-176); locking the flag to production mode or failing fast when an insecure configuration is detected would prevent token leakage.
- Attachment downloads echo the original filename into `Content-Disposition` without sanitizing quotes or newlines (apps/api/src/attachments/attachments.controller.ts:122) even though uploads merely trim whitespace (apps/api/src/attachments/attachments.service.ts:68-106); malicious filenames can inject headers or alter response handling, so always escape/encode the filename before use.
- Uploads persist buffers via `fs.writeFileSync`, which blocks the Node event loop for the entire file (apps/api/src/attachments/attachments.service.ts:95-98); streaming the uploaded stream to disk or handing off writes to a worker thread keeps the API responsive under concurrent uploads.
- Global validation (ValidationPipe + ApiExceptionFilter) and the CSRF guard ensure inputs are checked and CSRF headers/cookies are compared timing-safely before business logic runs, reducing exposure to invalid payloads (apps/api/src/main.ts:18-24; apps/api/src/common/csrf.ts:10-55).

## Code Quality & Performance
- The `update` and `schedule` controller actions rebuild patch objects and diff tracking field-by-field, duplicating logic twice inside the same file (apps/api/src/todos/todos.controller.ts:178-210;289-312); a shared helper or service-owned diff generator would reduce repetition and keep audit deltas consistent when fields are added.
- `useTodos` reruns `refresh` on every filter/sort/date change (effect beginning at apps/web/app/hooks/useTodos.ts:348) and also on every storage-event broadcast (`notifyTodoUpdate` at apps/web/app/hooks/useTodos.ts:60 triggering the listener at apps/web/app/hooks/useTodos.ts:364), so UI tweaks or cross-tab signals immediately fire new fetches even when data is already current; debouncing or caching query keys would cut redundant requests.
- Audit list helpers parse `details` with `JSON.parse` for every row returned, making paging through logs repeatedly execute string deserialization (apps/api/src/audit/audit.service.ts:147/207/272); storing the column as JSONB or handling parsing lazily would ease CPU pressure on high-volume audit queries.

## Maintainability & Documentation
- Change-tracking logic for todo updates/schedules is duplicated (`changes` objects around apps/api/src/todos/todos.controller.ts:202 and apps/api/src/todos/todos.controller.ts:289) with no shared helper; moving diff construction into the service or a reusable util keeps audit payloads in sync as more fields are introduced.
- The README/quick-start doc explains the stack but omits key environment variables such as `COOKIE_SECURE`, `CSRF_COOKIE_NAME`, and `OCR_WORKER_BASE_URL`, which are referenced across the application (apps/api/src/auth/auth.controller.ts:171; apps/api/src/common/csrf.ts:10; apps/api/src/ocr/ocr.service.ts:207); documenting them avoids guesswork for new deployments.

## Optimisable Areas
- The per-todo child-count queries inside `enrichWithChildCounts` can be replaced with a single aggregate query (apps/api/src/todos/todos.service.ts:64;167), eliminating the heavy `Promise.all`/`SELECT` loop and giving predictable latency for wide result sets.
- Cross-tab broadcasts always trigger a `refresh`, so toggling filters or sorting before the server has new data still reissues the same query (apps/web/app/hooks/useTodos.ts:60;348-364); adding a cache key or dirty flag prevents needless round trips.
- Audit helpers parse `details` via `JSON.parse` on every row returned by the three list endpoints, which duplicates JSON parsing work for each page (apps/api/src/audit/audit.service.ts:147/207/272); storing `details` as JSONB and letting Drizzle deliver the parsed object would make scanning logs faster.

## Overall Assessment
- Strength: The global validation/CSRF hooks plus the consistent audit logging in controllers (e.g., todo creates/updates) provide a reliable, observable platform that guards against malformed inputs and records every state change (apps/api/src/main.ts:18-24; apps/api/src/common/csrf.ts:10-55; apps/api/src/todos/todos.controller.ts:115-232).
- Weakness: Attachment handling still reflects user filenames directly in headers and blocks the event loop when persisting uploads, while the todo listing layer issues repeated child-count queries; addressing those issues will simultaneously raise security posture and scalability (apps/api/src/attachments/attachments.controller.ts:122; apps/api/src/attachments/attachments.service.ts:95-98; apps/api/src/todos/todos.service.ts:64; apps/api/src/todos/todos.service.ts:167).
